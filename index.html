手机web监听摇一摇事件

发表于 2017-06-02 | 分类于 Javascript ， js小知识点 |

问题描述：

手机打开网页时，想在用户摇一摇手机后做些动画效果。

解决方案：

  • html5提供了几个新的DOM事件来获得设备物理方向及运动的信息，包括：陀螺仪、罗盘及加速计。

    第一个DOM事件是deviceorientation，其提供设备的物理方向信息，表示为一系列本地坐标系的旋角。

    第二个DOM事件是devicemotion，其提供设备的加速信息，表示为定义在设备上的坐标系中的卡尔迪坐标。其还提供了设备在坐标系中的自转速率。

    第三个DOM事件是compassneedscalibration，其用于通知Web站点使用罗盘信息校准上述事件

取消上层div的鼠标事件，让下层div处理

发表于 2017-04-28 | 分类于 Javascript ， js小知识点 |

问题描述：

两个div，上层div是一张logo图片，我希望所有鼠标事件由下层处理，上层不拦截鼠标事件。

解决方案：

  • 我想要的效果就类似于鼠标事件穿透上层div直接点到下层，试了很多方法没搞定，没想到一个css设置就搞定了。
  • 样式采用绝对布局，关键设置是pointer-events: none;
  • 这是下层div就可以处理全部的鼠标事件，上层不会拦截鼠标事件了。

为什么要用settimeout而不用setinterval

发表于 2017-04-28 | 分类于 Javascript ， js小知识点 |

问题描述：

使用setInterval做循环时，如果执行的代码在间隔时间内没有执行完，最后会挤在一起执行。

解决方案：

  • 原因：setInterval不会考虑代码的执行时间，有可能代码还没有执行完就再次调用了代码，这样所有没执行完的代码都会排在事件队列末尾，cpu空闲时一起执行了剩余的代码。（只是个人理解不一定准确）

  • 我想要的效果是间隔固定时间后执行一段代码，代码执行完后，间隔相同的时间再次执行这段代码。

  • 可以用settimeout实现
